## 🎨 ISPIRAZIONE: LEVIATHAN MARKETING COMPANY

### Analisi Completa Pagine Servizi Leviathan

#### **1. DIGITAL EXPERIENCE PAGE - Struttura Vincente**

**Hero Section:**

```
"Creiamo esperienze digitali che uniscono design e performance: 
progettate per distinguerti e far crescere il tuo business."
```

**Punti di Forza:**

- ✅ Value proposition in una riga (chiara, concreta)
- ✅ Benefit immediato ("distinguerti e far crescere")
- ✅ No buzzword inutili

**Sezione "In cosa possiamo aiutarti":**

```
SEO, velocità, user journey e obiettivi di business 
sono parte del progetto fin dall'inizio.
```

**Pattern identificato:**

- 5-6 bullet points con titolo bold + descrizione breve
- Focus su QUANDO e COME, non solo COSA
- Ogni punto inizia con un verbo d'azione

**Case Study Section:**

- Badge con settore (`[ TRAVEL | Headless eCommerce ]`)
- Risultato concreto in 1-2 righe
- NO metriche vaghe, solo numeri specifici

**Tabella Comparativa LEVpod™:**

- Confronto a 3 colonne
- Righe: Team \& costi, Time-to-market, Regia \& KPI, Evoluzione, Operatività
- Icona info `i` per dettagli (UX pattern intelligente)

**FAQ Section:**

- Domande cliente-centriche (iniziano con "Posso...", "Come...", "Quanto...")
- Risposte brevi (2-3 righe max)
- Rassicurazioni concrete (es: "No perdita SEO", "Sì proprietà codice")

### Cosa Funziona Molto Bene (Da Replicare)

**✅ Copy Principles:**

1. **Concreto > Generico** - "10.45% CTR" non "ottimi risultati"
2. **Benefit immediato** - Prima riga dice cosa otterrà il cliente
3. **Workflow trasparente** - 4 step numerati con descrizione
4. **Tabella comparativa** - Mostra valore vs alternative
5. **FAQ cliente-centriche** - Rispondono a obiezioni reali
6. **Testimonial con ruolo** - Nome + Azienda + Ruolo per credibilità

**✅ Struttura Pattern:**

```
Hero (Value Prop)
    ↓
In Cosa Ti Aiutiamo (5-6 punti)
    ↓
Perché Sceglierci (6 pilastri)
    ↓
Case Study con Numeri
    ↓
Come Lavoriamo (Workflow 4 step)
    ↓
Soluzioni (Pacchetti/Servizi)
    ↓
Testimonial
    ↓
FAQ
    ↓
CTA Finale
```

## 1️⃣6️⃣ ANIMAZIONI CARD SERVIZI (STILE LEVIATHAN)

### Implementazione Scroll Animation

**Opzione 1: Intersection Observer (Consigliata per MVP)**

```javascript
// cards-animation.js
const observerOptions = {
  threshold: 0.2,
  rootMargin: '0px 0px -100px 0px'
};

const observer = new IntersectionObserver((entries) => {
  entries.forEach((entry, index) => {
    if (entry.isIntersecting) {
      // Delay incrementale per effetto cascata
      setTimeout(() => {
        entry.target.classList.add('is-visible');
      }, index * 100);
    }
  });
}, observerOptions);

// Osserva tutte le card
const cards = document.querySelectorAll('.service-card, .pillar, .case-study-card');
cards.forEach(card => observer.observe(card));
```

**CSS corrispondente:**

```css
/* Stato iniziale */
.service-card,
.pillar,
.case-study-card {
  opacity: 0;
  transform: translateY(40px) scale(0.98);
  transition: all 0.6s cubic-bezier(0.16, 1, 0.3, 1);
}

/* Stato visibile */
.service-card.is-visible,
.pillar.is-visible,
.case-study-card.is-visible {
  opacity: 1;
  transform: translateY(0) scale(1);
}

/* Stagger delay per effetto cascata */
.service-card:nth-child(1) { transition-delay: 0ms; }
.service-card:nth-child(2) { transition-delay: 100ms; }
.service-card:nth-child(3) { transition-delay: 200ms; }
.service-card:nth-child(4) { transition-delay: 300ms; }
.service-card:nth-child(5) { transition-delay: 400ms; }
.service-card:nth-child(6) { transition-delay: 500ms; }

/* Rispetta prefers-reduced-motion */
@media (prefers-reduced-motion: reduce) {
  .service-card,
  .pillar,
  .case-study-card {
    transition-duration: 0.01ms !important;
    transition-delay: 0ms !important;
  }
}
```
